#  clinic_management System

A comprehensive clinic management system built with **Next.js 16**, **TypeScript**, **MongoDB**, and **Tailwind CSS**.

## Getting Started

First, run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

This project uses [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) to automatically optimize and load [Geist](https://vercel.com/font), a new font family for Vercel.

## Cloudinary Logo Upload

The clinic settings page uploads the clinic logo to Cloudinary using a server-signed upload.

Add these environment variables in `.env.local`:

```bash
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
```

The browser asks `/api/cloudinary/signature` for a short-lived signature before uploading the image directly to Cloudinary.

## Enforce Unique User Emails

If you have older data, run the index cleanup script once after setting `MONGODB_URI`:

```bash
npm run sync:user-email-index
```

The script drops the old tenant-scoped email index, checks for duplicate emails across all users, and then rebuilds the unique email index.
